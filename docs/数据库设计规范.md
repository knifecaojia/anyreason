## 1. 设计规约 (Design Conventions)

### 1.1 命名规范 (Naming Convention)

* **表名** ：使用 **复数形式** ，小写下划线分隔（`snake_case`）。例：`projects`, `video_prompts`。
* **主键** ：统一命名为 `id`，类型为 `UUID`。
* **外键** ：`[单数表名]_id`。例：`project_id`, `scene_id`。
* **索引** ：`idx_[表名]_[字段名]` 或 `uq_[表名]_[字段名]` (唯一索引)。
* **时间字段** ：统一使用 `created_at` (创建时间) 和 `updated_at` (更新时间)。

### 1.2 核心策略 (Core Strategy)

* **ID 双轨制** ：
  * **物理主键 (PK)** ：使用 `UUID v4`，用于系统内部关联，确保全局唯一且无序。
  * **业务编码 (Code)** ：对外展示/人类阅读字段（如 `role.name`、`permission.code`），并以唯一约束保证一致性。
* **JSONB 使用原则** ：
  * 易变结构存储为 JSONB（如 `audit_logs.meta`）。
  * 核心查询字段必须提升为实体列（如 `audit_logs.action`、`permissions.code`）。
* **软删除 (Soft Delete)** ：权限与审计模块当前不启用软删除；如未来涉及合规与恢复需求，可为核心表增加 `deleted_at`。

## 3. 物理模型详解 (Physical Schema)

### 3.1 角色表 (roles)

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |
| --- | --- | --- | --- | --- |
| `id` | `UUID` | ✅ | `gen_random_uuid()` | 物理主键 |
| `name` | `VARCHAR(64)` | ✅ | - | 角色名，唯一 |
| `description` | `TEXT` | - | - | 角色描述 |
| `created_at` | `TIMESTAMP` | ✅ | `now()` | 创建时间 |
| `updated_at` | `TIMESTAMP` | ✅ | `now()` | 更新时间 |

索引：
* `UNIQUE INDEX uq_roles_name (name)`
* `INDEX idx_roles_name (name)`

---

### 3.2 权限表 (permissions)

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |
| --- | --- | --- | --- | --- |
| `id` | `UUID` | ✅ | `gen_random_uuid()` | 物理主键 |
| `code` | `VARCHAR(128)` | ✅ | - | 权限编码，唯一（建议 `system.users` 这种点分层级） |
| `description` | `TEXT` | - | - | 权限描述 |
| `created_at` | `TIMESTAMP` | ✅ | `now()` | 创建时间 |
| `updated_at` | `TIMESTAMP` | ✅ | `now()` | 更新时间 |

索引：
* `UNIQUE INDEX uq_permissions_code (code)`
* `INDEX idx_permissions_code (code)`

---

### 3.3 用户-角色关联表 (user_roles)

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |
| --- | --- | --- | --- | --- |
| `id` | `UUID` | ✅ | `gen_random_uuid()` | 物理主键 |
| `user_id` | `UUID` | ✅ | - | 关联用户（FK -> `user.id`） |
| `role_id` | `UUID` | ✅ | - | 关联角色（FK -> `roles.id`） |
| `created_at` | `TIMESTAMP` | ✅ | `now()` | 创建时间 |

索引：
* `UNIQUE INDEX uq_user_roles_user_role (user_id, role_id)`
* `INDEX idx_user_roles_user (user_id)`
* `INDEX idx_user_roles_role (role_id)`

### 3.5 系统审计日志表 (audit_logs)

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |
| --- | --- | --- | --- | --- |
| `id` | `UUID` | ✅ | `gen_random_uuid()` | 物理主键 |
| `actor_user_id` | `UUID` | - | - | 操作人（FK -> `user.id`，可为空） |
| `action` | `VARCHAR(128)` | ✅ | - | 事件动作（如 `role.create`、`user.roles.set`） |
| `resource_type` | `VARCHAR(64)` | - | - | 资源类型（如 `role`、`user`） |
| `resource_id` | `UUID` | - | - | 资源 ID（如角色/用户 UUID） |
| `success` | `BOOLEAN` | ✅ | `true` | 成功标记 |
| `request_id` | `VARCHAR(64)` | - | - | 请求 ID（用于串联应用日志与审计记录） |
| `ip` | `VARCHAR(64)` | - | - | 客户端 IP |
| `user_agent` | `TEXT` | - | - | UA |
| `meta` | `JSONB` | ✅ | `{}` | 额外信息（不存敏感数据） |
| `created_at` | `TIMESTAMP` | ✅ | `now()` | 创建时间 |

索引：
* `INDEX idx_audit_logs_actor (actor_user_id)`
* `INDEX idx_audit_logs_created_at (created_at)`
* `INDEX idx_audit_logs_action (action)`

## 4. 枚举字典 (Enumeration Dictionary)

### 4.1 系统权限建议集（示例）

* `system.users`: 用户管理（用户列表、分配角色）
* `system.roles`: 角色与权限管理（维护权限字典、角色授权）
* `system.audit`: 审计日志查看
