## 1. 设计规约 (Design Conventions)

### 1.1 命名规范 (Naming Convention)

* **表名** ：使用 **复数形式** ，小写下划线分隔（`snake_case`）。例：`projects`, `video_prompts`。
* **主键** ：统一命名为 `id`，类型为 `UUID`。
* **外键** ：`[单数表名]_id`。例：`project_id`, `scene_id`。
* **索引** ：`idx_[表名]_[字段名]` 或 `uq_[表名]_[字段名]` (唯一索引)。
* **时间字段** ：统一使用 `created_at` (创建时间) 和 `updated_at` (更新时间)。

### 1.2 核心策略 (Core Strategy)

* **ID 双轨制** ：
  * **物理主键 (PK)** ：使用 `UUID v4`，用于系统内部关联，确保全局唯一且无序。
  * **业务编码 (Code)** ：对外展示/人类阅读字段（如 `role.name`、`permission.code`），并以唯一约束保证一致性。
* **JSONB 使用原则** ：
  * 易变结构存储为 JSONB（如 `audit_logs.meta`）。
  * 核心查询字段必须提升为实体列（如 `audit_logs.action`、`permissions.code`）。
* **软删除 (Soft Delete)** ：权限与审计模块当前不启用软删除；如未来涉及合规与恢复需求，可为核心表增加 `deleted_at`。

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |

### 3.8 资产绑定表 (asset_bindings)

说明：资产可绑定到剧集/分场/分镜三层之一；通过约束保证每条绑定仅指向一个层级目标。

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |
| --- | --- | --- | --- | --- |
| `id` | `UUID` | ✅ | `gen_random_uuid()` | 物理主键 |
| `asset_entity_id` | `UUID` | ✅ | - | 资产实体（FK -> `assets.id`，on delete cascade） |
| `asset_variant_id` | `UUID` | - | - | 资产变体（FK -> `asset_variants.id`，on delete set null） |
| `episode_id` | `UUID` | - | - | 绑定目标：剧集（FK -> `episodes.id`） |
| `scene_id` | `UUID` | - | - | 绑定目标：分场（FK -> `scenes.id`） |
| `shot_id` | `UUID` | - | - | 绑定目标：分镜（FK -> `shots.id`） |
| `state` | `JSONB` | ✅ | `{}` | 绑定状态（镜头内摆放/服饰差异等扩展） |
| `created_at` | `TIMESTAMP` | ✅ | `now()` | 创建时间 |

约束：
* `CHECK ck_asset_bindings_single_target`：`episode_id/scene_id/shot_id` 恰好一个非空
* `UNIQUE uq_asset_bindings_episode_asset (episode_id, asset_entity_id)`
* `UNIQUE uq_asset_bindings_scene_asset (scene_id, asset_entity_id)`
* `UNIQUE uq_asset_bindings_shot_asset (shot_id, asset_entity_id)`

索引：
* `INDEX idx_asset_bindings_asset (asset_entity_id)`
* `INDEX idx_asset_bindings_episode (episode_id)`
* `INDEX idx_asset_bindings_scene (scene_id)`
* `INDEX idx_asset_bindings_shot (shot_id)`

### 3.9 LLM 虚拟 Key 表 (llm_virtual_keys)

说明：为每个用户签发 LiteLLM Virtual Key 的元数据。明文 Key 仅在创建时返回一次，数据库仅保存加密后的 token 与可展示前缀。

| **字段名** | **类型** | **必填** | **默认值** | **描述/约束** |
| --- | --- | --- | --- | --- |
| `id` | `UUID` | ✅ | `gen_random_uuid()` | 物理主键 |
| `user_id` | `UUID` | ✅ | - | 归属用户（FK -> `user.id`，on delete cascade） |
| `purpose` | `VARCHAR(32)` | ✅ | `'default'` | 用途标识（如 `default` / `tool_xxx`） |
| `litellm_key_id` | `VARCHAR(128)` | - | - | LiteLLM 返回的 key 标识（若存在） |
| `key_prefix` | `VARCHAR(16)` | ✅ | - | 展示用前缀（例如前 12 位） |
| `key_hash` | `VARCHAR(64)` | ✅ | - | token 的 SHA256（十六进制），用于去重与关联 |
| `encrypted_token` | `BYTEA` | - | - | token 的加密存储（Fernet），用于吊销/轮换时调用 LiteLLM 删除接口 |
| `status` | `VARCHAR(16)` | ✅ | `'active'` | 状态（`active/revoked/expired`） |
| `created_at` | `TIMESTAMP` | ✅ | `now()` | 创建时间 |
| `revoked_at` | `TIMESTAMP` | - | - | 吊销时间 |
| `expires_at` | `TIMESTAMP` | - | - | 过期时间 |
| `last_seen_at` | `TIMESTAMP` | - | - | 最近一次被使用时间（预留） |

约束：
* `CHECK ck_llm_virtual_keys_status`：`status` 必须为 `active/revoked/expired`

索引：
* `UNIQUE INDEX uq_llm_virtual_keys_key_hash (key_hash)`
* `INDEX idx_llm_virtual_keys_user (user_id)`
* `INDEX idx_llm_virtual_keys_user_status (user_id, status)`
* `INDEX idx_llm_virtual_keys_created_at (created_at)`
