# 创作工坊（无限画布）方案A - Phase 2（连线参考 + 生成接入）

> **文档状态：** Draft  
> **当前版本：** v0.1.0  
> **最后更新：** 2026-02-20  
> **维护人：** 研发（Trae IDE 协作）

---

## 0. 背景与目标

Phase 1 已完成 `/studio` 入口替换与无限画布 UI 骨架（左悬浮菜单 + 右侧 Tab Pages + ReactFlow 画布 + VFS 持久化）。

Phase 2 聚焦两件事：

1. **连线参考收集**：让用户通过“连线”表达参考关系，系统在目标 LLM 节点的提示词面板中汇总展示参考来源（缩略图/标题），并支持一键跳转到源节点。
2. **生成接入**：让 MediaNode（LLM 节点）可以直接调用图片/视频生成 API，并将结果写回节点数据，同时支持“参考图”随请求传入。

---

## 1. 非目标（Phase 2 不做）

- 不做公共资产池（跨项目可见）相关能力。
- 不做多人协作与冲突解决（仍以单人创作为主）。
- 不做完整的资产变体落库与归档（只做生成结果回写节点 + 可选归档到 VFS）。
- 不做复杂的连线类型体系（仅以 `reference` 关系为主，其他关系留到后续）。

---

## 2. 数据结构约定

### 2.1 连线语义（Edge）

- 继续使用 `edge.data.relation = 'reference'` 表达参考关系
- 默认新建连线即为 `reference`
- 后续如要扩展 `extend/compare` 等关系，优先扩展 `edge.data.relation` 枚举，而非自定义 `edge.type`

### 2.2 MediaNode（LLM 节点）核心数据（前端 ViewModel）

建议在节点 data 中维护以下字段（仅示意，可与现有实现逐步对齐）：

- `prompt`、`negativePrompt`
- `modelConfigId?`（或 provider+model）
- `aspectRatio`、`duration`
- `output`：
  - `status: 'idle' | 'running' | 'success' | 'error'`
  - `url?: string`（生成结果地址）
  - `errorMessage?: string`
  - `createdAt?: string`
- `referenceCache?`（可选）：仅用于 UI 加速的缓存，不作为真相源；真相源应由 edges + nodes 运行时计算

## 3. 交互设计

### 3.1 参考来源收集（运行时）

触发条件：
- 用户选中 MediaNode 且满足 Focus 条件（`selected && viewport.zoom >= 0.6`）
- 显示屏幕空间提示词面板（Portal 浮层，尺寸不随 zoom 变化）

收集规则：
- 取所有 `target = mediaNodeId && edge.data.relation === 'reference'` 的边
- 对每条边读取 `sourceNode`，按类型提取“可作为参考的媒体”

来源节点到参考媒体的映射（Phase 2 版本）：
- **AssetNode**：
  - 优先：asset 详情返回的缩略图 / 代表图（需要补充前端获取资产详情的逻辑）
  - 退化：如果缺少可用图片，仅展示标题占位，仍可作为“语义参考”（但不参与 `images[]`）
- **MediaNode（image）**：
  - 如果有 `output.url`，作为参考候选
- **MediaNode（video）**：
  - Phase 2 不强制抽帧；若后端/网关已返回可用封面图，则可用封面图作为参考候选，否则仅展示标题占位
- **ReferenceNode（故事板条目）**：
  - 通常无图片，仅展示文本（shot_code/描述/对白），不参与 `images[]`

### 3.2 提示词面板中的“参考来源”呈现

UI 内容（位于提示词面板顶部）：
- 横向滚动的参考来源列表（chip/card）
  - 缩略图（可用时）
  - 标题（节点标题/资产名/shot_code）
  - 移除按钮（仅移除“参考边”，不删除节点）
- 操作：
  - 点击卡片：跳转到源节点（画布居中 + 选中）
  - 点击移除：删除对应 reference edge

提示词面板的屏幕空间行为（必须满足）：
- 面板尺寸与字号固定，不随 zoom 变化
- 面板位置只与“节点屏幕坐标”相关：平移/缩放时更新锚点位置
- 当目标节点离开可视区域时：
  - 面板自动隐藏（或显示“已离开视野”的轻提示），避免浮层悬空

### 3.3 连线创建与编辑

Phase 2 默认：
- 用户从节点端口拉线创建边，边默认 `relation='reference'`

可选增强（如实现成本低可在 Phase 2 内做）：
- 边的右键菜单：切换 relation（reference/extend/compare），其中只有 reference 会参与“参考来源收集”

## 4. 生成接入（图片/视频）

### 4.1 接口对齐（现有能力）

前端通过 Next BFF 调用：
- `POST /api/ai/image/generate` → 后端 `POST /api/v1/ai/image/generate`
- `POST /api/ai/video/generate` → 后端 `POST /api/v1/ai/video/generate`

请求体（后端 schema 现状）：
- `binding_key`（默认 image/video）
- `model_config_id`（可为空）
- `prompt`
- `resolution?`（image）
- `duration`、`aspect_ratio`（video）
- `images: string[]`（参考图列表，元素为 data URL 字符串）

### 4.2 参考图打包策略（images[]）

目标：将“参考来源卡片”中可用的图片打包到 `images[]`，随生成请求发送。

Phase 2 推荐策略（按可控性排序）：
1. **优先使用 VFS 可下载的图片**：通过 `/api/vfs/nodes/{id}/download` 拉取二进制并转 data URL（最稳定，避免跨域）
2. **允许使用同源可访问的图片 URL**：`fetch(url) -> blob -> data URL`
3. **无法获取时降级**：仅作为“语义参考”展示，不进入 `images[]`，并在面板提示“部分参考图不可用”

约束：
- `images[]` 上限：建议默认 0~4 张（可配置），避免请求过大或模型侧失败
- 每张图大小控制：建议在前端对过大图片做压缩（如后续引入压缩工具，再实现）

### 4.3 生成流程（MediaNode）

状态机：
- idle → running → success/error

成功后：
- 写回 `output.url`（用于节点预览）
- 写回 `output.createdAt`

失败后：
- 写回 `output.errorMessage`（面板内展示，避免 alert 打断创作）

## 5. 生成结果归档（VFS，Phase 2 建议实现为“可选”）

目标：让生成结果不只是“一个 URL”，而是可控地落到 VFS，便于后续被引用、下载、复用。

建议两阶段：

### 5.1 Phase 2.0（最小可用）

- 画布 JSON 中只记录 `output.url`
- 不做上传归档

### 5.2 Phase 2.1（归档增强）

前提：补齐 Next BFF 对 `POST /api/v1/vfs/files/upload` 的代理（`/api/vfs/files/upload`）。

归档流程：
- 生成成功后，前端尝试下载 `output.url` 并上传到 VFS
- 建议目录：
  - `/创作工坊/画布输出/${canvas_id}/`
- 上传成功后：
  - 将 `output.file_node_id`（或等价字段）写回节点
  - 预览优先使用 `/api/vfs/nodes/{id}/download`

失败策略：
- 归档失败不影响生成成功（只提示“归档失败”并保留 url）

## 6. 验收清单（Phase 2）

- [ ] 可创建 reference 连线（默认 relation=reference）
- [ ] 选中 MediaNode 且 zoom≥0.6 时显示提示词面板（屏幕空间，不随缩放）
- [ ] 提示词面板展示“参考来源”列表（来自 incoming reference edges）
- [ ] 点击参考来源可跳转源节点（居中 + 选中）
- [ ] 从提示词面板移除参考来源会删除对应 reference edge
- [ ] MediaNode 可调用图片生成 API 并在节点内展示结果
- [ ] MediaNode 可调用视频生成 API 并在节点内展示结果
- [ ] 生成请求会携带可用的参考图（images[]，至少能覆盖 VFS 图片来源）
- [ ] 生成失败会在提示词面板内展示错误信息（不使用 alert 打断）

## 7. 风险与缓解

| 风险 | 影响 | 缓解措施 |
|---|---|---|
| 参考图来自第三方 URL 跨域/鉴权失败 | 参考图无法进入 images[] | 优先走 VFS；URL 转 data URL 失败则降级为“语义参考” |
| images[] 体积过大导致请求失败 | 生成失败率提升 | 限制张数、限制单图大小，必要时引入压缩 |
| 浮层提示词面板锚点抖动 | 体验差 | 仅在 viewport/resize 时更新锚点；增加节流/去抖 |
| 视频无封面图无法作为参考 | 参考链路断裂 | Phase 2 只做占位；后续引入抽帧或网关返回封面图 |
