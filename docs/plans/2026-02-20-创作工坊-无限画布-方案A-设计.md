## 0. 产品定位

### 0.1 核心理念：以画布为核心

**画布是主角，一切围绕画布服务。**

- 无限画布占据视觉中心，是用户的创作空间
- **左侧悬浮图标菜单**，hover 时展开，不干扰创作
- **右侧 Tab Pages 面板**（故事板/资产库/详情）可折叠，不占用画布空间
- 没有固定的"底部提示词栏"，每个节点管理自己的提示词

### 0.2 入口与数据绑定

**画布是独立的，不与剧本/剧集强绑定。**

- 创作工坊入口替换现有 `studio`（不新增/保留 `workshop`）
- 用户进入创作工坊后，默认进入空白画布创作
- 剧本/剧集通过「故事板」Tab 动态引入
- 切换剧本/剧集 = 切换故事板数据源，不切换画布
- 画布可以混合引用多个剧本的素材

### 0.3 与 Tapnow 的差异

| 维度 | Tapnow | 本项目 |
|------|--------|-------|
| **布局** | 画布核心 + 可折叠侧栏 + 悬浮菜单 | 相同 |
| **背景** | 点状网格 | 相同 |
| **节点类型** | 通用图片/视频生成节点 | 资产节点 + 媒体节点 + 文本节点 + 参考节点 |
| **连线** | 无特殊语义 | 参考关系（收集参考来源） |
| **提示词** | 节点内嵌 | LLM 节点 Focus 时显示屏幕空间提示词面板（锚定节点位置） |

## 1. 设计目标与边界

### 1.1 目标（要做到）

- **无限画布为核心**：画布占据主要视觉空间，背景为点状网格
- **左侧悬浮菜单**：图标菜单，hover 展开，包含工具、设置等
- **右侧可折叠 Tab Pages**：故事板 Tab + 资产库 Tab + 详情 Tab，点击折叠/展开
- **节点独立提示词**：LLM 节点 Focus 时显示屏幕空间提示词面板（锚定节点位置），非 LLM 节点无提示词
- **灵活数据源**：故事板可动态切换剧本/剧集，不绑定画布
- **连线参考收集**：通过连线建立参考关系

### 1.2 非目标（明确不做）

- 不做三栏固定布局。
- 不做底部统一提示词栏。
- 不做与剧本/剧集的强绑定入口。
- 不引入新的白板库（统一用 ReactFlow）。

### 2.2 左侧悬浮菜单

**默认态**：只显示图标列（约 48px）
**Hover 态**：展开显示文字标签

**菜单项：**
- 📁 文件（导出画布、导入画布）
- ➕ 添加节点（文本、媒体、资产引用）
- 🔗 生成设置（默认模型、画幅等）
- ⚙️ 设置（主题、快捷键）
- ❓ 帮助

### 2.3 右侧 Tab Pages 面板（可折叠）

**折叠态**：只显示一条窄条，包含 Tab 图标
**展开态**：宽度约 280px，包含完整的 Tab 内容

**Tab 结构：**
- **Tab 1「故事板」**：剧本选择器 + 场景列表 + 镜头列表
- **Tab 2「资产库」**：项目资产（Phase 2 才引入公共资产池/跨项目可见）
- **Tab 3「详情」**：Inspector（选中节点 / 选中故事板条目 / 选中资产条目）

### 2.4 画布背景

**点状网格背景：**
- CSS 实现：`background-image: radial-gradient(circle, #ccc 1px, transparent 1px)`
- 点间距：20px
- 点颜色：随主题变化（亮色模式 #ccc，暗色模式 #444）
- 网格不随画布缩放（固定视觉大小）：背景放在 ReactFlow 外层容器（屏幕空间），不要放在随 viewport transform 的图层里

## 3. 节点类型与交互

### 3.1 节点分类

**LLM 节点（需要调用 AI 生成）**：
- MediaNode（图片/视频生成）
- 有 Focus 态提示词面板

**非 LLM 节点（不需要调用 AI）**：
- TextNoteNode（文本笔记）
- AssetNode（资产引用）
- ReferenceNode（故事板引用）
- 无 Focus 态提示词面板

### 3.2 节点两态显示

**Compact（未聚焦）**：
- 所有节点类型
- 显示缩略图 + 标题
- 占用空间小

**Focus（聚焦）**：
- 选中 + zoom 足够大时触发
- **非 LLM 节点**：显示完整内容，无提示词面板
- **LLM 节点**：显示完整内容 + 提示词面板（屏幕空间浮层，锚定节点位置）

### 3.3 LLM 节点的提示词面板

**位置**：屏幕空间浮层（Portal），锚定到节点位置，不随节点缩放
**触发**：节点 Focus 时自动展开（选中 + zoom 足够大）
**行为约束：**
- 面板尺寸与字号固定（不随 zoom 变化）
- 面板位置跟随节点（随平移/缩放更新锚点位置），但面板本体不做 transform 缩放
**内容**：
- 参考来源（从连线收集的缩略图列表）
- 提示词输入框
- 负面提示词输入框
- 模型选择（下拉）
- 画幅/比例选择
- 生成按钮（图片/视频）

```
┌─────────────────────────────────┐
│  🎬 视频生成                     │  ← 节点标题
│  ─────────────────────────────  │
│  [预览区域：生成结果缩略图]        │  ← 预览
│  ─────────────────────────────  │
│  📎 参考来源: [张三] [李四] [+]   │  ← 参考来源（Focus时显示）
│  ─────────────────────────────  │
│  提示词:                         │
│  [________________________]     │  ← 提示词输入
│  负面: [__________________]     │
│  模型: [kling-v1 ▼]  16:9       │
│  [生成视频]                      │
└─────────────────────────────────┘
```

### 3.4 非 LLM 节点（以 AssetNode 为例）

```
┌─────────────────────────────────┐
│  👤 张三                         │  ← 资产名称
│  ─────────────────────────────  │
│  [资产缩略图]                     │  ← 资产图片
│  类型: 角色                       │
│  变体: [默认 ▼]                   │  ← 变体选择（Focus时显示）
│  ─────────────────────────────  │
│  （无提示词面板）                  │
└─────────────────────────────────┘
```

## 4. 右侧 Tab Pages 详情

### 4.1 Tab 1：故事板

**顶部：剧本选择器**
- 下拉选择：当前项目的剧本/剧集
- 切换剧本 = 切换故事板数据源
- 不影响画布内容

**内容区：场景列表**
- 按 scene 分组
- 每个 scene 显示：scene_code + 描述摘要
- 展开 scene 显示镜头列表
- 每个 shot 显示：shot_code + 描述 + 对白（如有）

**操作：**
- 点击 shot → 在右侧「详情」Tab 显示详情
- 拖入画布 → 创建 ReferenceNode
- 复制文本 → 复制到剪贴板

### 4.2 Tab 2：资产库

**顶部：切换按钮**
- [项目资产]

**内容区：资产列表**
- 按类型分组：角色 / 场景 / 道具 / 特效
- 每个资产显示：缩略图 + 名称 + 变体数
- 支持搜索

**操作：**
- 拖入画布 → 创建 AssetNode
- 点击 → 在右侧「详情」Tab 显示详情

### 4.3 面板折叠/展开

**折叠态**：
- 宽度：48px
- 显示：Tab 图标（故事板/资产库/详情）
- 点击图标 → 展开面板

**展开态**：
- 宽度：280px
- 显示：完整内容
- 点击折叠按钮 → 折叠面板

### 4.4 Tab 3：详情（Inspector）

**用途**：承载“点击查看详情”的稳定落点，避免把详情塞进悬浮菜单。

**展示优先级：**
- 选中画布节点 → 显示节点详情（基础信息、引用信息、动作入口）
- 未选中节点但点击故事板/资产条目 → 显示条目详情（支持一键拖入画布）

**LLM 节点（MediaNode）特殊说明：**
- 提示词面板为屏幕空间浮层，不放在详情 Tab 内
- 详情 Tab 显示：生成历史、归档信息、引用来源列表（可跳转源节点）

## 7. 数据模型

### 7.1 画布文档（CanvasDocument）

```typescript
interface CanvasDocument {
  version: 1;
  canvasId: string; // 画布唯一 ID（不绑定剧本/剧集）
  name: string; // 画布名称
  reactflow: {
    nodes: Node[];
    edges: Edge[];
    viewport: { x: number; y: number; zoom: number };
  };
  nodePayloads: Record<string, NodePayload>;
  storySource?: {
    scriptId: string;
    episodeId: string;
  }; // 可选：故事板数据源
  updatedAt: string;
}

type NodePayload = 
  | TextNoteNodeData 
  | MediaNodeData 
  | AssetNodeData 
  | ReferenceNodeData;
```

### 7.2 节点数据结构

**TextNoteNodeData**
```typescript
interface TextNoteNodeData {
  type: 'text-note';
  title: string;
  content: string; // Markdown
  tags: string[];
}
```

**MediaNodeData（LLM 节点）**
```typescript
interface MediaNodeData {
  type: 'media';
  title: string;
  fileNodeId?: string; // VFS 文件 ID（生成后绑定）
  mediaType: 'image' | 'video';
  // 提示词面板数据
  prompt: string;
  negativePrompt: string;
  modelConfig: {
    provider: string;
    model: string;
  };
  aspectRatio: string;
  duration?: number; // 视频
  // 参考来源（从连线自动收集，不存储）
}
```

**AssetNodeData**
```typescript
interface AssetNodeData {
  type: 'asset';
  assetId: string; // Asset 表 ID
  name: string;
  assetType: 'character' | 'scene' | 'prop' | 'vfx';
  selectedVariantId?: string;
  thumbnailUrl?: string;
}
```

**ReferenceNodeData**
```typescript
interface ReferenceNodeData {
  type: 'reference';
  title: string; // shot_code
  content: string;
  sourceInfo: {
    storyboardId: string;
    sceneCode: string;
    shotCode: string;
    scriptId: string;
    episodeId: string;
  };
}
```

### 7.3 连线与参考收集

**连线语义：**
- 不依赖自定义 `edge.type`
- 使用 `edge.data.relation = 'reference'` 标记参考关系（渲染仍可用默认边）

**参考收集（运行时计算，不存储）：**
```typescript
function collectReferences(nodeId: string, edges: Edge[], nodes: Node[]) {
  const referenceEdges = edges.filter(e => e.target === nodeId && e.data?.relation === 'reference');
  return referenceEdges.map(e => {
    const sourceNode = nodes.find(n => n.id === e.source);
    return {
      nodeId: sourceNode.id,
      thumbnail: getNodeThumbnail(sourceNode),
      title: getNodeTitle(sourceNode),
    };
  });
}
```

## 9. 持久化策略

### 9.1 Phase 1：VFS 优先（本地兜底）

- 默认存储：VFS（复用现有 vfs API，不新造“workshop 专用持久化 API”）
- 文件位置建议：`/创作工坊/画布/${canvas_id}.json`
- 自动保存：debounce 800ms（减少频繁写入）
- 本地兜底：`localStorage['studio_canvas_draft_${id}']` 仅在网络失败时暂存，成功写入 VFS 后清理

### 9.2 Phase 2：多人协作/版本化（可选）

- VFS 文件级版本策略（例如保留最近 N 个版本或按时间归档）
- 冲突策略：以最后写入为准 + 提示用户手动复制/对比

- 结果：绑定到 MediaNode 的 fileNodeId

## 11. 实施路径

### Phase 0：设计确认（本文件）

- 对齐以画布为核心的布局
- 对齐节点分类（LLM vs 非LLM）
- 用户审批后进入实施

### Phase 1：画布核心 + 左侧菜单 + 右侧面板

**前端任务：**

### Phase 3：资产管理

**前端任务：**

## 13. 风险与缓解

| 风险 | 影响 | 缓解措施 |
