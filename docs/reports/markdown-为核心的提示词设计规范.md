# Markdown-first 提示词设计规范（面向内置工具）

## 背景与目标

在业务系统中，“内置工具”的职责通常同时包含两类需求：

- **主产物（Canonical Output）**：模型按提示词生成的 Markdown 文本，业务系统要能直接展示、复制、归档（所见即所得）。
- **可操作性（Operability）**：业务系统仍需要最小的稳定结构字段用于列表展示、检索过滤、资产绑定、溯源审计等。

本规范的目标是采用 **Markdown-first + 轻结构派生** 的方式，兼顾：

- Markdown 输出的灵活性（你想要的卡片模板、Visual Prompt、导演补全等都能完整保留）。
- 业务系统的稳定可操作性（至少能枚举资产条目，并用于后续绑定与流程自动化）。

## 核心原则（Checklist）

1) **单一主产物**：最终输出以 Markdown 为中心；不要在同一 prompt 里同时要求“仅输出结构化 JSON”。  
2) **可解析锚点最小化**：只规定少量固定锚点行，让系统能 best-effort 地抽取 name/keywords/episode 等字段。  
3) **失败可降级**：解析失败时也必须保留 raw Markdown 原文（业务系统仍可展示与归档）。  
4) **结构字段与业务一致**：锚点只覆盖系统真正需要且能承载的字段；其余信息全部留在 Markdown 正文中。  
5) **分隔符强约定**：卡片之间用固定分隔符，便于拆分成每个资产文件。  

## 推荐的 Markdown 卡片结构（最小锚点）

### 分隔符

每个卡片之间用一行分隔：

```
---
```

### 卡片标题（必须）

标题行必须包含“调用名”（以 `@` 开头），建议包含形态区分：

```
### 角色资产卡片：@叶辰_常态
```

> 说明：系统可把 `@叶辰_常态` 记录进 provenance 或用于后续绑定/引用。

### 固定锚点行（建议全部提供）

为了让系统能派生轻结构字段，建议卡片内包含以下 Key: Value 行（大小写不敏感，中文键名也可，但要固定）：

```
Name: 叶辰
Keywords: 复仇者, 黄阶元印师
FirstAppearanceEpisode: 2
```

规则：

- `Name` 必须是稳定资产名称（用于文件名/索引/绑定）。
- `Keywords` 用逗号/顿号分隔均可。
- `FirstAppearanceEpisode` 不确定可省略或留空。

### 其余内容（完全自由）

强烈建议用固定小标题来组织（方便后续增强解析器），但不做硬性限制：

- 视觉外观（Appearance）
- 性格演绎（Personality Mapping）
- AI 绘画提示词（Visual Prompt）
- 导演补全备注（Director’s Note）

## Prompt 设计建议

### 你应该写什么

- **输出形态约束**：卡片标题格式 + 分隔符 + 最小锚点行。
- **覆盖范围**：明确“从哪些信息源识别角色/道具/地点/特效”（出场人物、对白、旁白、动作描述、系统旁白等）。
- **补全策略**：当外观缺失时如何补全（配色、服装、体型、气质）。
- **反问禁令**：缺信息时禁止反问，必须给出导演级补全。

### 你不应该写什么（常见反模式）

- 同时要求：
  - “按以下 Markdown 模板输出”
  - “仅输出结构化 JSON / 仅输出结构化数据”

这会导致模型输出折中、字段缺失或格式不稳定，最终“看起来不听话”。

- 要求模型把整段 Markdown 作为 JSON 字符串输出（除非你接受转义/转码成本与偶发解析失败）。

## 轻结构映射（Markdown → 业务字段）

业务侧常用“轻结构 + details_md”模式（例如 AssetDocV2）：

- `type`：由工具/场景确定（character/prop/location/vfx）
- `name`：来自 `Name:` 行
- `keywords`：来自 `Keywords:` 行（分隔符拆分）
- `first_appearance_episode`：来自 `FirstAppearanceEpisode:` 行
- `details_md`：整张卡片 Markdown 原文（保持原样）
- `provenance`：工具补充（agent_code/version/tool_id/model/call_name 等）

> 关键点：**details_md 存的是原文，不是“结构化字段拼回 Markdown”。**  
> 结构字段只是为了业务可操作性（列表/检索/绑定）。

## 可直接复用的 Prompt 模板（角色提取）

将以下模板作为“工具额外指令”或提示词的一部分（按需替换字段）：

```
你是漫剧角色资产架构师。基于剧本提取角色，并以 Markdown 角色卡片输出。

输出规则：
1) 每个角色输出一张卡片
2) 卡片标题行固定为：### 角色资产卡片：@调用名
3) 每张卡片必须包含锚点行（Key: Value）：
   - Name: 角色名称
   - Keywords: 关键词1, 关键词2
   - FirstAppearanceEpisode: 首次出场集数（不知道就省略）
4) 每张卡片之间用一行 `---` 分隔
5) 不要输出 JSON，不要输出额外解释段落

补全要求：
- 当外观缺失时，必须导演级补全（配色/服饰/体型/气质），禁止反问。
- Visual Prompt 必须是英文，包含 A-pose/Neutral Background/Best Quality 等关键词。

覆盖要求：
- 尽量覆盖剧本中出现或明确暗示的重要角色；如有“出场人物”行，必须覆盖其名单。
```

## 验证清单（交付前自检）

- 能否直接把模型输出 Markdown 展示在 UI（不依赖解析）？
- 能否从 Markdown 锚点 best-effort 解析出 name/keywords/episode？
- 卡片是否按 `---` 正确拆分为多个资产文件？
- 即使解析失败，是否仍保留 raw Markdown（可归档、可复制）？
