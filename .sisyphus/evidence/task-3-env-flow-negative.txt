# TASK-3: Mismatches and Issues Found

## Mismatch #1: Missing Build Arg in compose.app.yml

**Location**: docker/compose.app.yml lines 115-118

**Expected**:
```yaml
frontend:
  build:
    args:
      NODE_BASE_IMAGE: ${NODE_BASE_IMAGE:-node:20-bookworm}
      NEXT_PUBLIC_API_BASE_URL: ${NEXT_PUBLIC_API_BASE_URL:-http://localhost:8000}
```

**Actual**:
```yaml
frontend:
  build:
    args:
      NODE_BASE_IMAGE: ${NODE_BASE_IMAGE:-node:20-bookworm}
```

**Impact**: NEXT_PUBLIC_API_BASE_URL is undefined at build time, so frontend uses fallback value (likely empty or defaults from code).

---

## Mismatch #2: Runtime ENV Too Late

**Location**: docker/compose.app.yml line 127

**Problem**: Setting `NEXT_PUBLIC_API_BASE_URL` as runtime environment variable has no effect on Next.js because:
- Next.js already completed `pnpm run build` at lines 22-28 of Dockerfile.prod
- The value was captured during build time
- Runtime ENV cannot change the baked-in value

**Evidence**:
- Dockerfile.prod line 28: `RUN pnpm run build` executes AFTER ARG/ENV on lines 22-23
- Any runtime ENV changes happen AFTER the build is complete
- Next.js public env vars are immutable at runtime

---

## Mismatch #3: Same Issue in docker-compose.deploy.yml

**Location**: docker/docker-compose.deploy.yml lines 183-184, 194

**Problem**: Same issue - build args missing NEXT_PUBLIC_API_BASE_URL, only runtime ENV is set.

**Additional Issue**: Line 191 shows `NODE_ENV: development` but this is a production deploy - should be `production`.

---

## Runtime-Only Variables That ARE Correctly Used

These variables are set at runtime only and don't need build args:
- `INTERNAL_API_BASE_URL: http://backend:8000` - Used for internal server-to-server calls
- `API_BASE_URL: http://backend:8000` - Could be used by client-side runtime
- `OPENAPI_OUTPUT_FILE` - Runtime file path
- `NEXT_PUBLIC_USE_MOCK_API: "false"` - Runtime-only flag
- `USE_MOCK_API: "false"` - Runtime-only flag
- `NEXT_SERVER_ACTIONS_ENCRYPTION_KEY` - Runtime key

---

## Verification Commands

To verify NEXT_PUBLIC_API_BASE_URL is baked:
```bash
# Inside container
grep -r "NEXT_PUBLIC_API_BASE_URL" /app/.next/static

# On host after build
docker run --rm anyreason-frontend grep -r "http://localhost:8000" /app/.next/static
```

---

## Risk Assessment

**High Risk**: Frontend is building with undefined/empty API URL, then runtime ENV is uselessly set. 
- Frontend API calls will fail or use wrong URL
- Redeploying with new API URL requires full rebuild anyway
- The runtime ENV setting creates false impression that URL can be changed without rebuild

**Medium Risk**: NODE_ENV=development in production deploy (line 191 in docker-compose.deploy.yml)
