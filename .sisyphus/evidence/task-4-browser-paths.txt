## 3. Nginx Proxy Configuration

From `docker/nginx/anyreason-https.conf`:

### 3.1 API Proxy (lines 66-80)
```
location /api/ {
    proxy_pass http://anyreason_backend/api/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_connect_timeout 60s;
    proxy_send_timeout 300s;
    proxy_read_timeout 300s;
}
```

### 3.2 WebSocket Proxy (lines 82-95)
```
location /api/v1/tasks/ws {
    proxy_pass http://anyreason_backend/api/v1/tasks/ws;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_connect_timeout 60s;
    proxy_send_timeout 3600s;
    proxy_read_timeout 3600s;
}
```

### 3.3 Frontend Root (lines 113-126)
```
location / {
    proxy_pass http://anyreason_frontend;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

### 3.4 Static Assets (lines 128-133)

```
location /_next/static/ {
    proxy_pass http://anyreason_frontend/_next/static/;
    expires 365d;

```

## 5. Conclusion

The frontend architecture correctly relies on:
1. Relative API paths that the browser resolves to the current origin
2. Nginx proxying /api/ requests to the backend
3. WebSocket proxy at /api/v1/tasks/ws with proper Upgrade headers
4. Frontend served at root / with static assets cached at /_next/static/

This design is sound - browser requests to /api/* route through nginx to backend, while the frontend is served separately. No hardcoded origins create cross-origin issues.
