function CreditsHistoryContent() {
  const { rows, isLoading, error } = useCreditHistory();
  
  if (isLoading) {
    return <LoadingSkeleton />;        // Show skeleton
  }
  
  if (error) {
    return <ErrorState message={error} />;  // Show error
  }
  
  if (rows.length === 0) {
    return <EmptyState message="暂无积分流水" />;  // Show empty
  }
  
  return <TransactionList rows={rows} />;  // Show list
}

================================================================================
PAGINATION SUPPORT - HONEST REPRESENTATION
================================================================================

IMPORTANT: The backend API `/api/v1/credits/my/transactions` only supports
`limit` parameter, NOT offset-based pagination. This is a backend limitation.

NEW HOOK: useCreditHistoryWithLoadAll(limit) - Provides full refresh, NOT true pagination.

API Contract:
--------------------------------------------------------------------------------
{
  rows: CreditHistoryRow[],      // All loaded rows (limited by limit param)
  isLoading: boolean,            // Initial load in progress
  isRefreshing: boolean,         // Refresh/load-more in progress
  error: string | null,
  balance: number | null,
  hasMore: boolean,              // Indicates more records MAY exist beyond limit
  refresh: () => Promise<void>,  // Re-fetch all data (honest behavior)
  loadMore: () => Promise<void>  // DEPRECATED - alias for refresh()
}

Usage (Honest):
--------------------------------------------------------------------------------
function HistoryWithLoadAll() {
  const { rows, hasMore, refresh, isRefreshing, error } = useCreditHistoryWithLoadAll(50);
  
  return (
    <>
      <TransactionList rows={rows} />
      {error && <ErrorState message={error} />}
      <button onClick={() => refresh()} disabled={isRefreshing}>
        {isRefreshing ? "刷新中..." : "刷新"}
      </button>
      {hasMore && <p className="text-textMuted text-xs">显示最新 50 条记录</p>}
    </>
  );
}

Limitation Note:
--------------------------------------------------------------------------------
The `hasMore` flag indicates if the API returned exactly `limit` rows,
suggesting there may be more. However, `loadMore` (or `refresh`) will simply
re-fetch the same `limit` rows - it cannot fetch the next page of data.

For TRUE offset-based pagination, the backend must add:
- `?offset=N` query parameter support
- Response that only includes rows AFTER the offset

This hook is still valuable for:
- Displaying a snapshot of recent transactions
- Refreshing to see new transactions
- Future-proofing (easy to wire up true pagination when backend supports it)
