## New Sections Added

### 1. Environment Precedence Rules
- Local Development (Host Machine): .env.local → .env → defaults
- Docker Local Stack: compose environment: block → env_file → defaults
- Production Deployment: CI/CD → compose environment → env_file → image defaults
- Table format with authoritative variables for each context

### 2. Build-Time vs Runtime Semantics
- Explained which variables are baked into Docker image at build time
- Explained runtime-only variables (INTERNAL_API_BASE_URL)
- Created variable scope summary table showing Build-Time | Runtime | Browser-Exposed

### 3. Forbidden Patterns
Five forbidden patterns documented:
1. Browser Localhost - NEVER set NEXT_PUBLIC_* to localhost
2. Docker Internal Hostname in NEXT_PUBLIC - browser can't resolve "backend"
3. Duplicated Fallback Chains - multiple layers hide misconfiguration
4. Mixing Build-Time and Runtime for Same Variable
5. Using .env in Production - secrets in git is compromised

### 4. Quick Reference Card
One-page summary table for quick lookup:
- Maps scenarios (local dev, Docker local, production, split-origin) to correct variable settings

## Key Guardrails Documented
- NEXT_PUBLIC_* variables leak to browser and require rebuild to change
- INTERNAL_API_BASE_URL is runtime-only, internal network accessible
- Browser should use relative paths (baseURL = "") by default
- Never use localhost or internal Docker hostnames in NEXT_PUBLIC_*
- Define authoritative source in ONE place only (no silent fallbacks)
