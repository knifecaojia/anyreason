## API Call Behavior

### Browser-Side (Client SDK)
The generated OpenAPI client (`app/openapi-client/client.gen.ts`) creates client without baseURL:
```typescript
export const client = createClient(createConfig<ClientOptions>());
```
This means browser-side calls use **relative URLs** (e.g., `/auth/jwt/login`), which are same-origin and proxied through Next.js.

### Server-Side (SSR)
Server-side API routes (e.g., `app/api/docs/route.ts`) use:
```typescript
function getApiBaseUrl() {
  return process.env.INTERNAL_API_BASE_URL || process.env.API_BASE_URL || process.env.NEXT_PUBLIC_API_BASE_URL || "http://localhost:8000";
}
```
With `INTERNAL_API_BASE_URL=http://backend:8000` set in compose, SSR calls go directly to backend container.
