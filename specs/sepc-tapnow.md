#### 1. 产品定义：人主导的 AIGC 创作工作台 (Human-Centric AIGC Creation Canvas)

**一句话描述：**
一个基于无限画布的 AI 漫剧创作系统，它以**创作者**为核心，将剧本分解后的故事板和资产库作为**参考材料**展示在侧边栏，画布是自由的创作空间。系统提供工具与参考，但**不强制建立剧本与创作的自动关联**——连线、提示词、生成决策全部由人来主导。

---

#### 2. 核心设计哲学（修正版）

**我们需要向团队传达以下四个核心理念：**

**A. 画布是创作空间，不是剧本映射**

* **理念：** 画布是空的，用户自由创建文本/图片/视频/资产节点。
* **解释：** 剧本分解后的故事板（Storyboard）和资产库（角色/场景/道具）作为**侧边参考区**存在，帮助编导理清叙事逻辑和素材。
* **实现：** 用户可以选择"从故事板/资产库拖入"作为参考，但不是强制。系统不自动预生成画布结构。

**B. 弱关联，人决策**

* **理念：** 系统不自动建立"节点 ↔ 故事板"的强绑定。
* **解释：** 连线表达"参考关系"：源节点作为目标节点的生成参考。目标节点自动从入边收集参考来源，显示在提示词栏。
* **关键点：** 提示词由人编写，系统可以**辅助建议**但不自动拼接。

**C. 叙事流作为参考，工作流由人主导**

* **理念：** 故事板的线性结构和资产库的分类结构在侧栏展示，但画布上的节点布局完全由人决定。
* **解释：** X轴/Y轴没有强制语义。创作者可以把"分镜1"和"分镜10"放在一起对比，也可以按自己的工作流组织节点。

**D. 资产可见性层级**

* **理念：** 资产分为"项目资产"和"公共资产"两个层级。
* **解释：** 项目资产仅当前项目可见；公共资产可跨项目共享。项目资产可以"标记为公共"，供其他项目使用。
* **关键点：** 创作产物可以"保存为资产变体"，回馈资产库，形成闭环。

---

#### 3. 核心功能需求描述（修正版）

**为了实现上述理念，具体的开发需求如下：**

**需求：左栏双 Tab 参考区**

* **Tab 1「故事板」**：
  * 展示基于剧本分解得到的 Storyboard（按场/镜头排序）。
  * 每个 shot 显示：shot_code + description（截断）+ dialogue（如有）。
  * 操作：点击查看详情 / 拖入画布生成 ReferenceNode / 复制文本。
* **Tab 2「资产库」**：
  * 顶部切换：[项目资产] | [公共资产]
  * 按类型分组：角色 / 场景 / 道具 / 特效
  * 每个资产显示：缩略图 + 名称 + 变体数量
  * 操作：拖入画布创建 AssetNode / 点击查看详情 / 标记为公共
  * 支持搜索

**需求：无限画布工作区（ReactFlow）**

* 支持中键拖拽平移、Ctrl + 滚轮缩放，带合理的缩放上下限（0.15 ~ 2.5）。
* 节点两态显示：
  * Compact（未聚焦）：只显示缩略信息（文本摘要/缩略图）
  * Focus（聚焦）：显示完整信息 + 底部提示词栏
* 节点类型：
  * **TextNoteNode**：自由文本笔记
  * **MediaNode**：绑定 VFS 文件（图片/视频）
  * **AssetNode**：引用后端资产库（支持选择变体）
  * **ReferenceNode**：从故事板拖入的参考卡片

**需求：连线表达参考关系**

* 类型：`reference`（参考）| `extend`（延展）| `compare`（对比）
* 目标节点自动从"参考"类型连线收集参考来源
* 参考来源显示在底部提示词栏的"参考来源"区域
* 点击参考来源缩略图可跳转到源节点
* 删除参考来源 = 删除对应连线

**需求：底部提示词栏（聚焦态）**

* 仅当节点聚焦时显示
* **参考来源区域**：显示所有通过"参考"类型连线连接的源节点缩略图
* 字段：提示词、负面提示词、模型供应商、画幅、比例、时长（视频）
* 操作：生成图片 / 生成视频（由人触发）

**需求：画布持久化**

* Phase 1：浏览器 localStorage 兜底
* Phase 2：VFS 持久化（`/创作工坊/EP001_canvas.json`）

**需求：资产可见性管理**

* 项目资产 vs 公共资产
* 项目资产可以"标记为公共"
* 创作产物可以"保存为资产变体"

#### 4. 用户交互场景（修正版）

**场景 1：使用资产作为参考生成视频**

1. 用户进入创作工坊，左栏切换到「资产库」
2. 找到角色"张三"和"李四"，分别拖入画布，创建两个 AssetNode
3. 从这两个 AssetNode 拉出连线（类型=参考）到目标位置
4. 在目标位置创建一个新的 MediaNode（视频生成节点）
5. 选中 MediaNode，底部提示词栏自动显示"参考来源"：张三和李四的缩略图
6. 用户编写提示词："张三和李四在皇宫打斗"
7. 点击"生成视频"
8. 系统调用 API，携带参考图 URL
9. 生成结果归档到 VFS，绑定到 MediaNode

**场景 2：创作产物回馈资产库**

1. 用户对某个 MediaNode 的生成结果很满意
2. 右键菜单 → "保存为资产变体"
3. 选择目标资产（如"张三"）
4. 填写变体信息（如"张三-古装版"）
5. 调用 API 创建变体
6. 资产库中可见新变体

**场景 3：使用公共资产**

1. 用户在「资产库」切换到"公共资产"
2. 浏览跨项目共享的资产
3. 拖入画布使用（创建引用，不复制资产）
4. 在自己的项目中使用公共资产作为参考

### 总结给团队的话（Pitch）

> **"我们是在做一个**以人为核心的创作工作台**。**
> 
> Tapnow 的无限画布解决了'怎么画出一张好图'的问题，
> 而我们要解决的是'**如何让编导在一个空间里自由组织创作素材、参考剧本和资产、生成图片/视频**'的问题。
> 
> 我们的核心设计原则是：**弱关联、人决策**。系统提供工具和参考，但不强制建立自动化的剧本-创作映射。连线表达参考关系，但提示词、生成决策全部由人来主导。
> 
> 同时，我们构建了**资产闭环**：资产库 → 画布创作 → 产物回馈资产库。创作不是孤立的行为，而是不断积累和复用的过程。"
