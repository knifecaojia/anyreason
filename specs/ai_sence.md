**AI_Scene 智能创作场景系统需求规格书**
 **版本** ：v1.0
 **关键词** ：PydanticAI、Scene、Agent、Toolchain、剧本创作、资产关联

---

## 1. 术语与概念澄清

为避免与剧本创作中的"场景（Scene，指故事发生的地点/场次）"混淆，本文档统一使用：

* **AI_Scene** ：AI 应用情景，指一套可运行的 AI 应用配置（包含 Agent 人格、工具箱、交互模式、输出格式）
* **Story_Scene** ：剧本场景，指故事中的具体场次（第几场、何地、何时），属于业务数据，非本系统配置对象

 **核心定义** ：AI_Scene 是 **面向创作流程的可复用 AI 能力单元** ，每个 AI_Scene 封装了一个完整的"人机协作工作间"。

## 2. 系统核心架构

### 2.1 AI_Scene 的三层结构

每个 AI_Scene 由以下三层构成，缺一不可：

**人格层（Persona）**

* 绑定的基础 Agent（内置专家或自定义 Agent）
* 系统提示词版本（可继承基础提示词并追加场景特定指令）
* 模型参数（温度、输出长度等）

**工具层（Toolbox）**

* **已启用的工具集合** （从工具市场勾选）
* **工具执行策略** （顺序执行、并行执行、条件执行、人工确认后执行）
* **工具链编排** （工具 A 的输出如何映射为工具 B 的输入）

**交互层（Interface）**

* 输入模式（对话框、文档上传、画布拖拽、表单填写）
* 观测深度（是否展示思考过程、是否展示工具调用详情、是否展示 Token 消耗）
* 输出格式（纯文本、结构化卡片、Markdown、JSON、可视化图表）

### 2.2 默认内置 AI_Scene：ChatBox（不可删除）

 **定位** ：系统的"通用创作伴侣"，作为所有 AI_Scene 的基线和 fallback。

 **配置固定性** ：

* **Agent** ：仅绑定一个可配置的 Main Agent（默认为"剧本通才"）
* **工具箱** ：仅保留 **只读查询类工具** （查询剧本、查询资产、查询历史记录）， **禁止绑定写入类工具** （防止误操作）
* **交互模式** ：纯对话流式输出，无复杂观测面板，保持极简

 **战略价值** ：

* 作为用户首次接触系统的入口（零配置即用）
* 作为其他 AI_Scene 的"调试模式"（当专用 Scene 出问题时，可切回 ChatBox 继续工作）
* 作为系统最低可用保障（即使其他 Scene 崩溃，ChatBox 保持可用）

## 3. 工具箱（Toolbox）体系设计

工具是 AI_Scene 与业务系统（剧本库、资产库）的 **唯一交互接口** ，必须原子化、可组合、可观测。

### 3.1 核心工具分类（必须内置）

#### A. 文本处理工具（认知层）

* **Script_Segmenter（剧本切分）**
  * 输入：长文本/完整剧本
  * 参数：切分策略（按场次标记/按字数/按情节转折点）
  * 输出：剧集数组（每集包含标题、内容、字数统计）
  * 特性：支持增量切分（追加新内容时只处理新增部分）
* **Content_Normalizer（内容规范化）**
  * 输入：原始剧本文本
  * 功能：统一场记格式、修正标点、识别并标注角色对话 vs 动作描述 vs 旁白

#### B. 资产入库工具（写入层）

* **Episode_Saver（剧集入库）**
  * 输入：切分后的剧集数据
  * 前置校验：检查集数冲突、标题重复
  * 执行：写入 Episode 表，建立与 Project 的关联
  * 返回：生成的 Episode ID 列表
* **Asset_Creator（资产创建）**
  * 细分四个子工具（或一个工具带类型参数）：
    * Character_Saver：角色入库（含名称、年龄、特征、首次出场集数）
    * Prop_Saver：道具入库（含名称、类型、重要程度、所属角色）
    * Location_Saver：场景地点入库（含空间描述、时代背景、氛围关键词）
    * VFX_Saver：特效标记入库（含特效类型、出现集数、难度评级）
  * 冲突处理策略：发现同名资产时，返回"疑似重复"标记，供 Agent 决策（合并/新建/更新）

#### C. 资产治理工具（逻辑层）

* **Asset_Deduplicator（资产去重）**
  * 算法：模糊匹配（名称相似度+特征相似度）
  * 策略：自动合并（高置信度）/ 人工确认（中置信度）/ 保留独立（低置信度）
  * 输出：合并报告（哪些资产被合并，原因是什么）
* **Asset_Merger（资产合并）**
  * 输入：两个或多个资产 ID
  * 执行：合并属性（保留最完整的描述）、统一命名、更新所有引用关系
  * 事务：确保关联的 Episode 中的引用同步更新

#### D. 关系绑定工具（关联层）

* **Episode_Asset_Binder（剧集-资产关联）**
  * 功能：建立 Episode 与 Asset 的引用关系（多对多）
  * 类型：出场（角色）、出现（道具/特效）、发生（地点）
  * 反向索引：自动更新 Asset 的"出场集数列表"
* **Consistency_Checker（一致性校验）**
  * 检查逻辑：扫描 Episode 内容中提及的资产名称，是否都存在对应的 Asset 记录
  * 输出：缺失资产清单（哪些名称被提到但没入库）

### 3.2 工具执行策略（工具链编排）

AI_Scene 编辑器必须允许用户配置 **工具调用策略** ：

 **执行模式** ：

* **Auto（全自动）** ：Agent 决定何时调用，用户无感知（适合成熟流程）
* **Confirm（确认后执行）** ：Agent 生成调用意图，用户点击"确认"后执行（适合关键写入操作）
* **Step_by_Step（单步模式）** ：每执行一个工具暂停，等待用户输入"继续"（适合教学/调试）

 **错误处理** ：

* **Fail_Fast（快速失败）** ：任一工具失败即停止整个 AI_Scene
* **Fail_Safe（容错继续）** ：标记失败工具，继续执行后续工具，最后汇总错误报告
* **Retry（自动重试）** ：网络类工具失败时自动重试 3 次

## 4. AI_Scene 编辑器设计（交互融合）

### 4.1 核心创新：编辑态与运行态融合

传统模式：先编辑配置 → 保存 → 跳转到新页面运行 → 发现问题 → 返回编辑

 **本系统要求** ： **编辑态内嵌实时沙盒** ，实现"边配边测"的融合体验。

 **三栏布局** ：

* **左栏：配置面板** （What）
* 选择基础 Agent（7大专家之一）
* 工具箱组装（双列表选择器：可用工具 vs 已选工具）
* 策略配置（执行模式、错误处理、观测级别）
* **中栏：Prompt 工坊** （How）
* 场景专属指令编辑（在基础 Agent 提示词后追加）
* 变量插补（插入 `{{project_name}}` 等上下文变量）
* **实时预览按钮** ：将当前配置临时实例化为 Agent，立即测试
* **右栏：沙盒观测台** （Test）
* 输入测试剧本片段
* **实时运行** ：使用当前编辑中的配置（未保存也可运行）
* **差异对比** ：对比修改前后的输出差异（A/B Test）
* **工具调用 trace** ：显示如果发布到生产环境，会调用哪些工具、传什么参数

### 4.2 工具箱可视化配置

 **不是代码，也不是节点图** ，而是 **"能力卡片"配置** ：

1. **工具市场视图** ：网格展示所有可用工具（图标+名称+一句话描述），悬停显示详细参数
2. **工作区投放** ：点击工具卡片投放到中央工作区，成为该 AI_Scene 的工具链一环
3. **连线逻辑（轻量级）** ：

* 简单的 **顺序标记** （1, 2, 3）表示工具调用优先级
* **条件徽章** ：在工具卡片上挂条件标签（如"仅当集数>5时"），而非画线
* **数据映射** ：点击工具间的连接点，配置"上一个工具的输出字段如何映射到下一个工具的输入参数"

### 4.3 版本与继承

 **继承机制** ：

* 基于内置 Agent 创建 AI_Scene 时，提示词显示为"基础（只读）+ 追加（可编辑）"的双层结构
* 当平台升级基础 Agent 提示词版本时，AI_Scene 可选择"跟随升级"或"冻结当前版本"

 **版本快照** ：

* 每次保存 AI_Scene，生成版本快照（v1, v2...）
* 可对比任意两个版本的配置差异（Prompt 差异、工具增减差异）
* 支持一键回滚到历史版本

## 5. Agent 定义与 AI_Scene 的融合设计

### 5.1 关系定位：Agent 是 Class，AI_Scene 是 Instance

 **Agent（专家定义）** ：

* 长期稳定的身份（角色专家、分镜专家）
* 通用能力描述（系统提示词、基础工具集）
* 类似于"一个训练有素的人"

 **AI_Scene（应用情景）** ：

* Agent 在特定上下文中的 **运行时配置** （Handle）
* 绑定具体工具链（给这个人配具体的工具箱）
* 设定交互范式（这个人是负责快速咨询还是深度加工）
* 类似于"这个人在特定工作室、使用特定工具、处理特定类型任务"

### 5.2 融合点：双向编辑

 **从 Agent 到 Scene** ：

* 在 Agent 管理界面，可以"基于该 Agent 创建新 AI_Scene"，直接进入 AI_Scene 编辑器，预填充该 Agent 的配置

 **从 Scene 到 Agent** ：

* 在 AI_Scene 编辑器中，如果发现基础 Agent 能力不够，可以"升级为基础 Agent 编辑"，修改底层提示词后自动返回（需权限控制）

### 5.3 运行时融合：动态 Agent 装配

当用户打开一个 AI_Scene 开始对话时：

1. **动态装配** ：系统根据 AI_Scene 配置，实时组装 PydanticAI Agent 实例

* 基础人格（来自 Agent 定义）
* 场景专属提示词后缀（来自 AI_Scene 配置）
* 启用的工具集（来自 AI_Scene 工具箱）

1. **上下文注入** ：将当前项目 ID、用户 ID、已选 Story_Scene（剧本场次）等作为 `deps` 注入 Agent
2. **观测代理包装** ：在 Agent 外层包装观测层，将思考过程、工具调用实时推送到前端（仅当 AI_Scene 配置为"深度观测"模式）

## 6. 典型 AI_Scene 示例（非默认 ChatBox）

### 示例 A：资产提取工坊（Asset Factory）

* **绑定 Agent** ：资产提取专家
* **工具箱** ：Script_Segmenter → Asset_Creator（四个子工具并行）→ Asset_Deduplicator → Episode_Asset_Binder
* **交互模式** ：上传剧本 → 显示切分结果（可人工调整切分点）→ 显示提取的资产卡片（可人工合并/编辑）→ 一键入库
* **观测级别** ：高（显示每个资产提取的置信度、显示去重决策理由）

### 示例 B：分镜设计师（Storyboard Designer）

* **绑定 Agent** ：分镜专家
* **工具箱** ：Episode_Asset_Binder（查询关联资产）→ Consistency_Checker（校验资产引用）→ Storyboard_Saver（保存分镜）
* **交互模式** ：选择剧本场次 → 显示该场次已有资产 → 对话式生成分镜 → 右侧实时预览分镜表格
* **特殊配置** ：必须开启"确认后执行"模式（分镜保存前需人工确认）

### 示例 C：剧本医生（Script Doctor）

* **绑定 Agent** ：剧本专家
* **工具箱** ：仅查询工具（查询角色弧光、查询剧情节奏数据）+ Consistency_Checker
* **交互模式** ：纯对话咨询，不绑定写入工具（只读分析）
* **用途** ：检查剧本逻辑漏洞（如"张三在第五集已经死了，为什么第八集又出现了？"）

## 7. 数据流与权限

### 7.1 数据隔离

* **AI_Scene 配置** ：平台级（内置）或 团队级（自定义），可共享模板
* **AI_Scene 运行实例** ：用户级（每个用户的对话历史、运行结果隔离）
* **业务数据（Episode/Asset）** ：项目级（通过工具接口操作，受原有业务权限系统控制）

### 7.2 工具调用审计

所有通过 AI_Scene 调用的工具，必须记录：

* 调用者（User ID）
* 调用场景（AI_Scene ID）
* 输入参数（用于追溯"为什么创建了这个角色"）
* 执行结果（成功/失败）
* Token 消耗与成本

 **目的** ：当发现资产数据异常时，可追溯是哪个 AI_Scene、哪次对话、哪句用户指令导致的。

## 8. 关键验收标准

1. **零配置启动** ：新用户进入系统，默认打开 ChatBox AI_Scene，无需任何配置即可开始对话
2. **五分钟创建** ：用户可以在 5 分钟内通过编辑器创建一个新的自定义 AI_Scene（选择 Agent、勾选工具、测试、发布）
3. **工具链可视** ：用户可以清晰看到当前 AI_Scene 会调用哪些工具，以及工具间的数据流向（无需看代码）
4. **沙盒验证** ：在 AI_Scene 编辑器内，可以不保存配置直接测试运行，看到工具调用的详细参数（确保不会误操作生产数据，沙盒模式工具不实际写入）
5. **渐进式放权** ：从 ChatBox（安全只读）→ 标准 Scene（确认后写入）→ 全自动 Scene（Auto 模式），权限逐步开放，风险可控
